# 智能校园自习室预约管理平台

本项目是 Java 程序设计实践课程的大作业项目，面向高校自习室预约与座位管控场景，提供统一登录、角色分流、在线选座、管理员维护、预约管控和数据报表等功能。系统目前已经实现管理员端与学生端两套工作界面，网站整体为中文界面。

## 项目简介

平台用于解决校园自习室占座、空位不明确、预约管理困难等问题。学生可以按校区、楼栋、楼层和自习室筛选空闲座位，自由选择起止时间后在线预约；管理员可以维护楼栋、自习室、座位排布、学生账号、预约状态和基础统计数据。学生也可以自行注册账号。

系统采用前后端分离架构：

- 前端使用 React 构建单页应用，提供中文交互界面和图形化座位选择。
- 后端使用 Spring Boot 提供 REST API，并通过 Spring Data JPA 访问 PostgreSQL 数据库。
- 数据库使用 PostgreSQL，保存账号、楼栋、自习室、座位与预约记录。

## 当前功能

### 登录页（电影级 Hero 首页）

- 全屏视频背景（4 个校园场景轮播：晨光自习区、静谧阅览区、深静自习室、暮色研学区），1000ms 淡入淡出切换。
- 液态玻璃（`liquid-glass`）拟态效果，带渐变边框伪元素。
- 浮动纹理遮罩层，3s 浮动动画。
- 深色主题自动适配：切换至"深静自习室"视频时内容文字自动过渡为深色。
- 底部 4 个场景切换按钮，带激活态样式和 1s 冷却锁。
- 注册功能以弹窗形式独立展示，不占用登录面板空间。

### 统一登录与角色分流

- 管理员和学生共用同一个登录入口（不预填默认密码）。
- 支持使用 **昵称** 或 **学号**（纯数字自动识别）+ 密码登录。
- 登录成功后根据账号角色进入不同工作台。
- 默认初始密码为 `123456`。
- 登录后支持修改密码和编辑个人资料。
- 学生可在登录页自行注册账号（填写昵称、密码、姓名、学号；邮箱、手机号、班级可选填）。

### 管理员端功能

管理员登录后进入深色玻璃拟态主题的"管理员工作台"，包含以下模块：

- **首页**：概览面板，显示自习室管理快捷入口、学生账号数、进行中预约、占用率，以及自习室使用率 TOP5。
- **实时看板**：查看运行概况指标，自习室座位热力图（**按时长加权**计算使用率），**违规情况饼状图**（从全量预约数据统计），**预约时段柱状图**（全部预约含已释放，排除已取消）。
- **自习室管理**：维护校区、楼栋、楼层、自习室信息，录入座位行列数并生成座位排布。
- **可视化座位编辑**：点击自习室"修改"按钮弹出编辑弹窗，左侧编辑自习室属性，右侧**点击座位切换启用/停用**，实时预览（API 不可用时自动本地生成座位）。
- **账号管理**：新增管理员或学生账号，修改资料，删除账号，重置密码，启用/禁用账号，加入/解除黑名单。
- **预约管控**：查看所有预约记录（**按创建时间倒序排列**），支持签到、释放座位、标记违规、取消预约。
- **数据报表**：查看指定日期的当日累计预约、当前预约、占用率、各自习室使用率（按时长加权）和预约时段分布柱状图（全量预约统计）。
- **系统设置**：配置单日最大预约次数、签到窗口时长、违规黑名单阈值、未签到/未签退违规时间等参数（API 不可用时加载默认值）。

### 学生端功能

学生登录后进入浅色玻璃拟态主题的"学生预约中心"，包含以下模块：

- **首页**：欢迎横幅，快捷操作卡片（预约选座、进行中预约数、自习室总数、今日预约数），今日预约列表。
- **预约选座**：按校区、楼栋、楼层、自习室筛选，**自由选择起止时间**，查看图形化座位图。
- **我的预约**：查看个人预约记录，支持**签到**（开始后30分钟内）、**签退**（随时释放座位）、取消预约。
- **操作确认弹窗**：签到、签退、取消预约均有确认对话框，防止误操作。
- **个人中心**：查看姓名、学号等只读信息，编辑**昵称、邮箱、手机号、班级**，修改密码。
- **黑名单提示**：被加入黑名单的学生账号会收到提示，暂不能继续预约。

### 注册账号

- 登录页提供"没有账号？去注册"入口，点击弹出独立注册弹窗。
- 注册需填写：昵称（必填）、密码（必填，至少6位）、姓名（必填）、学号（必填）。
- 班级、手机号、邮箱为选填项。
- 注册成功后自动创建学生角色账号，并跳转回登录页。
- 注册接口位于 `POST /api/auth/register`。

## 核心数据模型

当前后端主要包含以下实体：

- `UserAccount`：账号信息，包含登录名（昵称）、密码、真实姓名、角色（管理员/学生）、学号、班级、手机号、邮箱、启用状态、黑名单状态、违规次数等。
- `Building`：楼栋信息，包含校区、楼栋名称、楼层数和说明。
- `Room`：自习室信息，包含所属楼栋、楼层、名称、容量、开放时间、关闭时间、座位行列数等。
- `Seat`：座位信息，包含所属自习室、座位号、行列位置、是否启用、是否靠窗、是否有插座等。
- `Booking`：预约记录，包含预约学生、自习室、座位、日期、开始时间、结束时间、状态（预约中/已释放/已取消/违规）、签到状态等。

## 本地运行

### 1. 准备环境

请先确认本机已经安装：

- JDK 17 或更高版本
- Maven
- Node.js 与 npm
- PostgreSQL

### 2. 创建数据库

在 PostgreSQL 中创建数据库：

```sql
CREATE DATABASE studyroom_db;
```

后端数据库配置位于：

```text
study-room-booking-backend/src/main/resources/application.properties
```

当前配置示例：

```properties
server.port=8081
spring.datasource.url=jdbc:postgresql://localhost:5432/studyroom_db
spring.datasource.username=postgres
spring.datasource.password=123456
spring.jpa.hibernate.ddl-auto=update
```

### 3. 启动后端

```powershell
cd study-room-booking-backend
mvn spring-boot:run
```

后端默认运行在：

```text
http://localhost:8081
```

API 基础路径为：

```text
http://localhost:8081/api
```

### 4. 启动前端

打开新的终端窗口：

```powershell
cd study-room-booking-frontend
npm install
npm start
```

前端默认运行在：

```text
http://localhost:3000
```

浏览器访问该地址后，可以使用默认管理员账号或学生账号登录，也可以自行注册新账号。

## 主要 API

### 认证与注册

- `POST /api/auth/login`：昵称或学号 + 密码登录。
- `POST /api/auth/register`：学生自助注册（昵称、密码、姓名、学号为必填；班级、手机号、邮箱可选填）。
- `PUT /api/auth/change-password`：修改密码。

### 账号管理

- `GET /api/users`：查询账号列表。
- `POST /api/users`：新增账号（管理员操作）。
- `PUT /api/users/{id}`：修改账号信息。
- `PUT /api/users/{id}/profile`：修改个人资料（昵称、邮箱、手机号、班级）。
- `DELETE /api/users/{id}`：删除账号。
- `PUT /api/users/{id}/reset-password`：重置密码。
- `PUT /api/users/{id}/blacklist`：加入或解除黑名单。

### 系统设置（管理员）

- `GET /api/admin/settings`：获取系统设置参数。
- `PUT /api/admin/settings`：更新系统设置（单日预约次数、签到窗口、违规阈值等）。

### 楼栋与自习室

- `GET /api/buildings`：查询楼栋列表。
- `POST /api/buildings`：新增楼栋。
- `PUT /api/buildings/{id}`：修改楼栋。
- `DELETE /api/buildings/{id}`：删除楼栋。
- `GET /api/rooms`：查询自习室列表。
- `POST /api/rooms`：新增自习室并生成座位。
- `PUT /api/rooms/{id}`：修改自习室。
- `DELETE /api/rooms/{id}`：删除自习室。
- `GET /api/rooms/{id}/seats?date=YYYY-MM-DD&startTime=HH:MM&endTime=HH:MM`：按日期和时间段查询座位状态。
- `GET /api/rooms/{id}/all-seats`：获取自习室全部座位（用于编辑）。
- `PUT /api/rooms/{roomId}/seats/{seatId}`：更新单个座位属性（启用/停用）。

### 预约管理

- `GET /api/bookings`：查询全部预约记录。
- `GET /api/bookings/user/{userId}`：查询指定用户的预约。
- `POST /api/bookings`：创建预约。
- `DELETE /api/bookings/{id}`：取消预约（已签到不可取消）。
- `PUT /api/bookings/{id}/cancel`：取消预约。
- `PUT /api/bookings/{id}/release`：释放座位。
- `PUT /api/bookings/{id}/check-in`：预约签到（仅限预约开始后30分钟内）。
- `PUT /api/bookings/{id}/check-out`：签退（已签到后可随时释放座位）。
- `PUT /api/bookings/{id}/no-show`：标记违规。

### 数据报表

- `GET /api/reports/overview`：运行概况。
- `GET /api/reports/room-usage?date=YYYY-MM-DD`：各自习室使用率（统计当日所有非取消预约）。
- `GET /api/reports/time-slots?date=YYYY-MM-DD`：各小时预约数分布。
- `GET /api/reports/no-show-stats`：违规分类统计（未签到 / 未签退）。

## 主要业务规则

- 只有学生账号可以创建预约，管理员账号不能预约座位。
- 学生可以在登录页自行注册（必填：昵称、密码、姓名、学号；选填：班级、手机号、邮箱）。
- 被禁用或加入黑名单的学生账号不能预约。
- 预约日期不能早于当前日期。
- 预约结束时间必须晚于开始时间。
- 同一学生单日最多允许 3 条有效预约。
- 同一座位在时间冲突时不能被重复预约。
- **签到规则**：仅限预约当天，在预约开始时间至开始后30分钟内可签到。已签到不可重复签到。
- **签退规则**：已签到的预约可随时签退，签退后座位释放。
- **取消限制**：已签到的预约不可取消，需先签退。
- **违规规则**：分为两类——
  - **Type 1（未签到）**：预约开始超过30分钟未签到自动标记违规。
  - **Type 2（未签退）**：已签到但超过结束时间30分钟仍未签退自动标记违规。
- 违规次数达到3次后，账号会被自动加入黑名单。
- 取消预约、释放座位、签到和违规均更新预约状态，不会直接删除历史记录。
- 楼层筛选根据所选校区/楼栋动态显示实际有房间的楼层（如负一楼显示为”B1层”）。
